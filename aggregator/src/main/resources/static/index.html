Demo of Solace Messaging as a Service in the Cloud

Using IBM Bluemix with CloudFoundry as the PaaS

The goal of this demo is to illustrate Solace Messaging-as-a-Service integration with CloudFoundry using an IBM Bluemix environment. Solace is uniquely positioned to provide a single unifying digital backbone for applications no matter how the connect whether itâ€™s in the public cloud, hybrid cloud, and private cloud. This demo is an example of a worker offload pattern and is composed of the following parts:

  1. A RESTful Web Application receives incoming requests. In this demo, the job requests are described in terms of amount of work, rate to send the work to the Backend applications, and the min and maximum delay the Backend system should use to simulate the work.
  2. The web application sends the requests to a non-exclusive queue on a Solace VMR which is running as a VM in IBM Bluemix.
  3. Backend worker applications are bound to the Solace queue and process work requests. Backend applications can be horizontally scaled using CloudFoundry CLI. Increasing the scale will increase the overall processing throughput of the system.
  4. Once done the work, the Backend applications respond to the request. The Web Application tracks status for each job and keeps statistics which is displayed in the global stats and summary table below.
  5. The Web App and Backend Applications depend on a Solace Messaging Service. This service is provided within CloudFoundry by a Solace Demo Service broker which is itself a CloudFoundry application.
  6. All of the applications are stages and started through the regular CloudFoundry admin CLI.
